Following are steps for configuring the development workstation environment:

1. Install Git Client

The installer for the Git client which is compatible with Visual Studio Code can be downloade from https://git-scm.com/.
Optionally, install GitHub Desktop to make it easier to create and manage repositories that are cloned locally from Git.

2. Install node.js

The installer for node.js can be downloaded from https://nodejs.org.

3. Install Visual Studio Code

Installer can be downloaded from https://code.visualstudio.com/download.

Configure global git settings for credential helper and email address

For Windows installations, it's probably best to use the Windows Credential Store to store your Git login credentials, so you don't have to enter your username and password every time you synchronize with the remote repository. The following will configure git to use the windows credential helper: 
git config --global credential.helper wincred

Configuring for GitHub remote Server

GitHub uses the configured email address to identify the account you are are authenticating:

Using your personal private email address that is associated with the github account: 

git config --global user.email email@address
Using github-provided no-reply email address that is associated with the github account: 
git config --global user.email userid+loginName@users.noreply.github.com

Configuring user name

The user name determines what user name is displayed when repository changes are checked in (this is not the same as authentication account info) 
git config --global user.name user name
