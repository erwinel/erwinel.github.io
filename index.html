  2. Navigate to System Definition Scripts - Background
  3. Copy and paste the text from the textbox below into the Run Script into the ServiceNow page text box and click the "Run Script" button.
    • If the script is successful, you will see *** Script: There were no errors. toward the end of the output.
  4. Once the script execution has completed, navigate away from the current page and refresh your browser to view the changes that were just made.
    Note: This script updates several properties, which causes the internal property cache to get updated. For this reason, it can take a few minutes for this script to execute.
Note: In order to provide proper non-repudiation during the setup process, separate ServiceNow user accounts should be created for each person who will be involved with the setup process.

For each administrative account to be added:

  1. Ensure your roles are elevated to security_admin.
  2. Navigate to User Administration Users and click New.
  3. Fill in form fields:
    • Login Name:
      • Cloud Instance: Use format of last name + first initial + _da.
      • On-Prem Instances: Use same login name as the user's administrative Active Directory account.
    • Time Zone: Set time zone to be the same as the system time zone.
  4. Add users to Roles
    • If necessary, re-open the user's record from User Administration Users.
    • Under the Roles tab at the bottom of the page, click the Edit button to modify the user's roles.
    • Select the admin and security_admin roles, move them to the right, and save changes.
  • When personal admin accounts have been created, these accounts should be used instead of the admin account whenever possible.
      Note: Typically, roles would be assigned through group memberships, and not directly assigned to individuals. This will happen once Active Directory synchronization is configured.

  1. Ensure your roles are elevated to security_admin.

    When elevating your roles, they do not remain elevated indefinitely. Therefore, it is a good idea to periodically re-elevate your role.
    Role elevations expire after a time span which is configured within the server. Also, role elevations may be lost if any network or system event causes the browser to re-negotiate login credentials.

  2. Verify and/or Modify plugin list (optional)

    The script which will be used bulk plugin activation (scripts\bulk_plugin_activate.js) contains a list of the IDs for the plugins to be activated. The plugin IDs are defined within ServiceNow. If any plugins have dependencies, those will be activated as well. Those which have already been activated will be ignored.

    Checking plugin IDs
    • Navigate to System Definition System Plugins to view available plugins.
    • The ID field/column contains the plugin ID that would listed within the scripts\bulk_plugin_activate.js file if it is to be activated.
    • The Requires field indicates which other plugins must be activated along with the current plugin.
    • The Status field indicates whether the plugin was already activated.

  3. Navigate to System Definition Scripts - Background

  4. Copy and paste the text from the textbox below into the Run Script into the ServiceNow page text box and click the "Run Script" button.

    This script may need to execute for an extensive duration. This script will continue activating plugins for at least 5 minutes or until they have all been activated.
    After 5 minutes has elapsed, the script will wait for the current activation to finish, and then stop. The purpose of this behavior is protect against any possible runaway conditions, should they happen (which is unlikely).
    Therefore, if all plugins were not activated, you will need to re-execute the script. The script will skip any plugins that have already been activated.
    Monitoring status of plugin activation
    While the background script is running, you may not be able to navigate in your current session until it is complete. To check the status of individual activations, you may need to open ServiceNow in a different browser.
    • Navigate to System Diagnostics Progress Workers to view the progress information. Each plugin activation will have a separate record.
    • The Status field/column indicates whether it is running, completed or waiting to execute.
    • The Completion code field/column indicates whether it succeeded or not.

  5. Repeat if necessary.

    If all plugins have been activated, you will see something like 

    All plugin activations completed and/or verified.
    Otherwise, you will need to repeat at step #3.

    If all plugins were not activated with the alotted time frame, something similar to this toward the end of the output: 

    *** Script: Activated plugin SLA Contract Add-on (com.snc.sla.contract2): Plugin activated/upgraded successfuly
*** Script: The following plugins were not activated before 5 minutes had elapsed:
Task Activities (com.snc.task_activity)

Tree map (com.snc.treemap)

Restore Deleted Records (com.snc.undelete)

Vendor Performance (com.snc.vendor_performance)

WebKit HTML to PDF (com.snc.whtp)

Re-run this script to continue.

    If all plugins were successfully activated, then you'll see something similar to this at the end of the output: 

    *** Script: Activated plugin Vendor Performance (com.snc.vendor_performance): Plugin activated/upgraded successfuly
*** Script: Activated plugin WebKit HTML to PDF (com.snc.whtp): Plugin activated/upgraded successfuly
*** Script: All plugin activations completed and/or verified.

    Details of script
    There are many plugins which should come activated out of the box. This script iterates through all plugin activation assumptions in order to validate those assumptions. If it finds any plugins which have not been activated, that may indicate that some changes occurred since this documentation was developed, which further implies that this document would need to be validated and/or updated.

    If any expected plugins were not activated, you'll a message similar to 

    *** Script: Plugin WebKit HTML to PDF (com.snc.whtp) was not activated (Status = inactive).

    Otherwise, you'll see a message that says something like 

    All 348 expected plugins were found to have been activated.
